
README — Честное Казино (лендинг MVP)
------------------------------------

Файлы в архиве:
- index.html      — основная страница
- styles.css      — стили
- script.js       — небольшой демонстрационный скрипт
- README.txt      — этот файл

Инструкция по быстрой публикации:
1) Размести файлы на любом статическом хостинге (Netlify, Vercel, GitHub Pages, любой shared/VPS).
   - Простой вариант: GitHub Pages
     a) Создай репозиторий и загрузь эти файлы в корень ветки main.
     b) Включи Pages в настройках репозитория, выбрав ветку main и папку root.
   - Другой быстрый вариант: Netlify — drag & drop папку.

2) Настрой форму записи:
   - В index.html в форме стоит placeholder action, который ведёт на Google Forms.
   - Самый простой вариант: создай Google Form с полями Имя, Контакт, Гео. Нажми "Отправить", выбери ссылку. В action формы вставь URL вида: https://docs.google.com/forms/d/e/ID/formResponse
   - Альтернатива: подключить сервисы типа Formspree или Make (Integromat) для записи в вашу CRM/Google Sheets.

3) Подключи домен (если нужно):
   - На Netlify/Vercel/GitHub Pages можно привязать кастомный домен (.live, .cam, .online и т.д.).
   - Настрой A/ CNAME записи на провайдера.

4) Тестирование:
   - Открой страницу, нажми "Простой демо-проверщик" — это клиентская симуляция.
   - Протестируй отправку формы и проверь, что данные приходят в Google Sheets.

5) Метрики и трафик:
   - Подключи Google Analytics / Яндекс.Метрику (вставь код в head section) или воспользуйся pixel'ами рекламных платформ.
   - Для A/B теста можно клонировать страницу и менять заголовок/призыв.

6) Что дальше:
   - Если конверсия хорошая (регистрации/записи), запускаем MVP демо commit-reveal (прототип сервера + клиент).
   - Если низкая — корректируем тексты/ассеты, повторяем тест.

Если хочешь — могу помочь:
- Подготовить Google Form и настроить прием данных в Google Sheets.
- Подготовить репозиторий и загрузить файлы на GitHub и включить Pages.
- Настроить Netlify deployment (drag & drop) и привязать домен.
